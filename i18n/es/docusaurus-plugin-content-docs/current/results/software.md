## Software y Flujos de Trabajo Reproducibles

AI-HPC4CST concede especial importancia a la ciencia abierta, la transparencia y la reproducibilidad computacional.

El software desarrollado en el proyecto respalda la optimización basada en simulación, las estrategias de búsqueda mejoradas con IA y la ejecución escalable en entornos HPC.

Cuando sea apropiado, el software del proyecto será:

- Controlado mediante sistemas de versionado  
- Accesible públicamente  
- Documentado y mantenido  
- Vinculado a las publicaciones asociadas  

---

## Principios Fundamentales

La estrategia de software de AI-HPC4CST se basa en los siguientes principios.

### 1. Desarrollo Abierto

Siempre que sea posible, el software se desarrolla en repositorios públicos bajo la organización GitHub CST-Modelling-Tools, promoviendo la transparencia y la mejora colaborativa.

### 2. Reproducibilidad

Los resultados científicos se acompañan de:

- Especificación clara de las versiones de software  
- Descripción de los entornos computacionales utilizados  
- Documentación de los flujos de trabajo de optimización  
- Referencias a datos de entrada o archivos de configuración cuando corresponda  

### 3. Modularidad e Integración

Las herramientas se diseñan para:

- Interoperar con marcos de simulación de sistemas termosolares de torre  
- Integrar algoritmos de optimización mejorados con IA  
- Operar eficientemente en entornos HPC  
- Soportar flujos de optimización escalables basados en simulación

## Desarrollos Actuales y Futuros
